## Authority & Citation Policy

### üîß Academic Search Tools

Two Python scripts are available for finding authoritative citations:

#### Primary: API-based search (recommended)

```bash

python scripts/academic_search.py "retrieval augmented generation" --limit 20
python scripts/academic_search.py "embeddings neural networks" --limit 20

```

**Workflow: Retrieve 20, Curate 3+**
1. Run search with `--limit 20` to get broad results sorted by citation count
2. Review abstracts and paper types
3. Select **minimum 3** papers (more is better):
   - 1√ó **Canonical paper** that introduced the concept
   - 1√ó **Survey/review** that synthesizes the field
   - 1√ó **Implementation/application** paper showing practical use
   - Additional papers welcome for comprehensive topics
4. Format selected papers with linked titles (dofollow)

**Features:**
- No captcha/Cloudflare issues
- Returns: title, authors, year, venue, citation count, DOI, abstract
- Sorted by citation count (most authoritative first)
- Sources indexed: arXiv, PubMed, ACL, IEEE, Springer, Elsevier, SSRN, etc.

#### Fallback: Browser-based search (when API misses niche papers)


### 1Ô∏è‚É£ Academic citations (selective, not decorative)

Include academic citations only when:

- The concept originates in academic literature (e.g., RAG, embeddings)
- There is a canonical paper
- The concept is often misrepresented
- It strengthens definitional precision

Format (with links for SEO/GEO):

> Lewis et al. (2020), "[Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks](https://arxiv.org/abs/2005.11401)", NeurIPS.

**Why link to sources?**
- **SEO**: External dofollow links to authoritative domains (arXiv, DOI) signal content quality and boost E-E-A-T
- **GEO**: AI models can verify claims by following links, making content more citable

‚ö†Ô∏è **Use dofollow (normal links) for academic citations** ‚Äî nofollow signals "I don't endorse this", which undermines your citations. Reserve nofollow for user-generated or untrusted content only.

Rules:
- **Foundational AI concepts** (RAG, embeddings, LLM): **Minimum 3 citations** (more is better) ‚Äî canonical paper + survey + implementation/application
- **Domain-specific terms** (tax-law, legal): 1-2 citations if applicable
- Must include: author + year + title (linked) + venue
- Link to: arXiv, DOI, Semantic Scholar, or official source
- No upper limit ‚Äî comprehensive topics benefit from more citations
- No vague "studies show"

Academic citations with dofollow links enhance both SEO authority and GEO extractability.

## GEO (Generative Engine Optimization) Requirements

### Content Guidelines for AI Search Extractability

‚ö†Ô∏è **Critical for GEO pickup by AI search engines (ChatGPT, Perplexity, Gemini):**

| ‚úÖ DO | ‚ùå DON'T |
|-------|----------|
| Write educational, factual content | Mention product names (Auryth TX) |
| Use domain-agnostic explanations | Write promotional/marketing copy |
| Keep examples generic ("user submits a question") | Use domain-specific examples ("tax question", "legal query") |
| Include **3+ dofollow citations** (more is better) | Use only 1-2 citations for foundational concepts |
| Link citations to arXiv/DOI (dofollow) | Use nofollow or unlinked references |
| Use **ASCII diagrams** for flows (not images) | Use PNG/SVG images for process flows |
| Explain "why it matters" for the domain generally | Tie relevance to specific company features |
| Build semantic clusters via related terms | Leave entries isolated |

**Why this matters**: AI search engines extract and cite content that appears authoritative and educational. Product mentions and narrow domain framing signal promotional content and reduce extraction likelihood.

```mdx
## Definition

Retrieval-Augmented Generation (RAG) is a technique that enhances large language models by retrieving relevant documents from a knowledge base before generating responses. This grounds the AI's output in factual, up-to-date information rather than relying solely on training data.

## Why it matters

RAG is essential for any AI system that needs to provide accurate, up-to-date information. In tax and legal domains, it enables retrieval of current legislation, rulings, and precedents before generating advice‚Äîensuring responses reflect the latest regulatory changes rather than outdated training data.

## How it works

```
Question ‚Üí Embed ‚Üí Search KB ‚Üí Retrieve docs ‚Üí Generate ‚Üí Response
    ‚îÇ                         ‚îÇ
    ‚îî‚îÄ‚îÄ‚îÄ vector similarity ‚îÄ‚îÄ‚îÄ‚îò
```

1. User asks a question
2. System retrieves relevant documents from knowledge base
3. LLM generates response using retrieved context
4. Response is grounded in authoritative sources

**‚ÑπÔ∏è GEO Note:** ASCII diagrams are extractable by AI search engines. PNG/SVG images are not.

## Common questions

**Q: How is RAG different from fine-tuning?**

A: Fine-tuning permanently modifies model weights with new data. RAG retrieves information at query time, making it easier to update and audit.

**Q: Can RAG hallucinate?**

A: RAG reduces hallucinations by grounding responses in retrieved documents, but quality depends on the knowledge base and retrieval accuracy.
```
